## 協力ゲーム

現実の社会では複数のプレイヤーが協力して**グループ**をつくることがある．

- 複数の企業が共同事業を行う
- 労働者が労働組合をつくる
- 複数の政党が連立政権をつくる
- 国や地域が共同で環境対策を行う

このような状況ではプレイヤー全員が協力するとは限らない．
一部のプレイヤーだけが協力する可能性も含めて，グループの形成と利得の分配を分析する必要がある．

```{tip} 定義（協力ゲーム）
プレイヤーが拘束力のある合意を結んで提携（協力関係）を形成でき，提携が獲得した利得の分配を分析するゲームを <span style="color:red">協力ゲーム</span> という．
```

```{tip} 定義（提携）
協力を目的としてつくられるプレイヤーのグループを <span style="color:red">提携</span> という．

プレイヤー全体の集合を $N$ とすると，提携は $N$ の部分集合 $S\subseteq N$ として表される．
プレイヤー全員からなる提携 $N$ を <span style="color:red">全員提携</span> という．
```

たとえば，プレイヤーがA，B，Cの3人であるなら，

$$
N=\{A,B,C\}
$$

である．空集合を除く提携は，

$$
\{A\},\ \{B\},\ \{C\},\
\{A,B\},\ \{A,C\},\ \{B,C\},\
\{A,B,C\}
$$

の7つである．

### 特性関数

提携を形成したときに，そのメンバーが共同で獲得できる利得を考える．

```{tip} 定義（特性関数）
各提携 $S\subseteq N$ に対して，その提携が獲得できる利得の総額 $v(S)$ を対応させる関数

$$
v:2^N\to\mathbb{R}
$$

を <span style="color:red">特性関数</span> という．通常，誰も参加しない提携の価値を $v(\varnothing)=0$ とする．

※ $2^N$：$N$の部分集合全体からなる集合族．$N$のべき集合という．
```

例）$v(\{A,B\})$：プレイヤーAとBが提携した場合に得られるプレイヤーAとBの利得の総額

```{tip} 定義（提携形n人ゲーム）
プレイヤー集合 $N$ ($|N|=n$) と特性関数 $v$ の組 $(N,v)$ を <span style="color:red">提携形n人ゲーム</span> または <span style="color:red">特性関数形n人ゲーム</span> という．
```

特性関数は提携が確保できる利得の総額だけを表し，提携を実現する戦略は指定しない．

```{tip} 定義（譲渡可能効用）
提携が獲得した利得を，共通の単位で測り，提携のメンバー間で自由に分配できるとき，その利得を <span style="color:red">譲渡可能効用</span> という．

この講義では，特に断らない限り譲渡可能効用を仮定する．
```

### 2人交渉問題

2人交渉問題は提携形n人ゲームの特殊な場合として解釈できる．
ここでプレイヤーA，Bの利得を$u_A$，$u_B$とする．

実現可能集合$U$は次を満たす．

$$
U = \{(u_A,u_B)| \ u_A+u_B \le v(\{A,B\})\}
$$

交渉の不一致点の利得$d$は次を満たす．

$$
d = (v(\{A\}), v(\{B\}))
$$

### 具体例：ベンチャー企業の起業

大学生A・B・Cの3人がインターネット関連のベンチャー企業を起業しようとしている．
それぞれが単独または共同で起業した場合の収益を次の特性関数で表す．

| 提携 $S$ | $v(S)$（万円） |
|---|:---|
| $\{A\}$ | 60 |
| $\{B\}$ | 40 |
| $\{C\}$ | 20 |
| $\{A,B\}$ | 200 |
| $\{A,C\}$ | 150 |
| $\{B,C\}$ | 100 |
| $\{A,B,C\}$ | 240 |

この表は，たとえばAとBが共同で起業すれば，2人で合計200万円を獲得できることを意味する．
特性関数は総額を示すだけであり，200万円をAとBがどのように分けるかは別の問題である．

### 優加法性

互いに重ならない2つの提携が別々に活動する場合と，1つに合併する場合を比較する．
特性関数$v$が満たすと良い性質として，次の**優加法性**がある．  
※ 優加法性：関数解析でも出てくる重要な概念

```{tip} 定義（優加法性）
互いに重ならない任意の提携 $S,T\subseteq N$ に対して，

$$
v(S)+v(T)\le v(S\cup T)
$$

が成り立つとき，特性関数 $v$ は <span style="color:red">優加法的</span> であるという．
優加法的な特性関数をもつゲームを <span style="color:red">優加法的ゲーム</span> という．
```

優加法的ゲームでは2つの提携が別々に行動するより，合併したほうが全体の利得は小さくならない．

ベンチャー企業の例では，

$$
v(\{A\})+v(\{B\})=60+40=100<200=v(\{A,B\})
$$

である．また，

$$
v(\{A,B\})+v(\{C\})=200+20=220<240=v(N)
$$

である．他の組合せについても同じ条件が成り立つため，このゲームは優加法的である．

```{note} 演習1：特性関数と優加法性
ベンチャー企業の起業ゲームについて，次の問いに答えよ．

1. AとCが提携したときの価値を答えよ．
2. BとCが別々に起業する場合の価値の合計と，2人で提携する場合の価値を比較せよ．
3. 提携 $\{A,C\}$ と提携 $\{B\}$ について，優加法性の不等式が成り立つことを確認せよ．
4. このゲームが優加法的であることを，全ての提携の組合せについて確認せよ．
```

## 利得の配分

提携形n人ゲームの分析手法（提携を実現する戦略・利得の分配）
- **協力ゲーム理論**：提携を実現する行動の組を**協力解**として定義し，それを公理（**パレート最適性**，**個人合理性**）によって基礎づける．
- **非協力ゲーム理論**：交渉プロセスをダイナミックなゲームとして定式化し，交渉ゲームの**ナッシュ均衡点**として協力解を求める．

以下，全員提携 $N$ が形成されたときの価値 $v(N)$ をプレイヤー間でどのように分けるかを考える．
プレイヤー $i$ の利得を $x_i$ とし，利得ベクトルを $x=(x_1,\ldots,x_n)$ と書く．

### 実現可能性とパレート最適性

```{tip} 定義（実現可能性）
利得ベクトル $x=(x_1,\ldots,x_n)$ が

$$
\sum_{i\in N}x_i\le v(N)
$$

を満たすとき，$x$ は全員提携 $N$ に関して <span style="color:red">実現可能</span> であるという．
```

```{tip} 定義（パレート最適性）
実現可能な利得ベクトル $x$ について，すべてのプレイヤーの利得を広義の意味で増加させ，少なくとも1人の利得を真に増加させる別の実現可能な利得ベクトルが存在しないとき，$x$ は <span style="color:red">パレート最適</span> であるという．

譲渡可能効用をもつ優加法的ゲームでは，パレート最適性は

$$
\sum_{i\in N}x_i=v(N)
$$

と表される．
```

### 個人合理性

プレイヤー $i$ は，誰とも協力しなくても単独で $v(\{i\})$ を獲得できる．
したがって，協力によって得る利得がこの値を下回るなら，全員提携に参加する理由がない．

```{tip} 定義（個人合理性）
実現可能な利得ベクトル $x$ が，すべての $i\in N$ について

$$
x_i\ge v(\{i\})
$$

を満たすとき，$x$ は <span style="color:red">個人合理性</span> を満たすという．
```

※ 個人合理性を満たす実現可能な利得は交渉問題における**交渉領域**に相当する．

```{tip} 定義（配分）
パレート最適性と個人合理性をともに満たす利得ベクトルを，協力ゲームの <span style="color:red">配分</span> という．
```

### 具体例：ベンチャー企業の配分

ベンチャー企業の起業ゲームでは，全員提携の価値は240万円である．
したがって，配分 $x=(x_A,x_B,x_C)$ は，

$$
x_A+x_B+x_C=240
$$

および

$$
x_A\ge 60,\qquad x_B\ge 40,\qquad x_C\ge 20
$$

を満たす．

たとえば，

$$
x=(100,80,60)
$$

は，利得の合計が240であり，全員が単独で得られる利得以上を受け取るため，配分である．

```{note} 演習2：配分の条件
ベンチャー企業の起業ゲームについて，

2. $x,y,z$ 以外に配分となる利得ベクトルの例を1つ挙げよ．
```

## コア

個人合理性は各プレイヤーが単独で離脱する可能性だけを考えている．  
しかし，複数のプレイヤーが一緒に離脱して別の提携をつくる可能性も考えることもできる．

```{tip} 定義（提携合理性）
実現可能な利得ベクトル $x$ が，すべての提携 $S\subseteq N$ について

$$
\sum_{i\in S}x_i\ge v(S)
$$

を満たすとき，$x$ は <span style="color:red">提携合理性</span> を満たすという．
```

ある提携 $S$ について

$$
\sum_{i\in S}x_i<v(S)
$$

であれば，提携 $S$ のメンバーは，現在の配分より全員の利得を増やす別の分配を実現できる．
このとき，提携 $S$ は現在の配分に不満をもち，離脱しようとする．

```{tip} 定義（コア）
提携形n人ゲームにおいて，提携合理性を満たす配分の集合を <span style="color:red">コア</span> と呼ぶ．

すなわちコアは

$$
\mathcal{C}(v)
=\left\{x\in\mathbb{R}^n\ \middle|\
\sum_{i\in N}x_i=v(N),\quad
\sum_{i\in S}x_i\ge v(S)\ \forall S\subseteq N
\right\}
$$

である．
```

コアに含まれる配分では，どの提携も自分たちだけで離脱して全員の利得を改善できない．
したがって，コアは**提携による離脱に対して安定な配分**を表している．

### 具体例：ベンチャー企業のコア

ベンチャー企業の起業ゲームのコアは，次の条件を満たす．

$$
x_A+x_B+x_C=240
$$

$$
\begin{gather}
x_A+x_B \ge 200,\\
x_A+x_C \ge 150,\\
x_B+x_C \ge 100,\\
x_A\ge 60,\qquad x_B\ge 40,\qquad x_C\ge 20.
\end{gather}
$$

全員提携の条件を用いると，2人提携の条件は各プレイヤーの利得の上限に書き換えられる．
たとえば，

$$
x_A+x_B\ge 200
$$

と $x_A+x_B+x_C=240$ から，

$$
x_C\le 40
$$

を得る．同様に，

$$
x_A\le 140,\qquad x_B\le 90
$$

である．したがって，コアは

$$
\begin{gather}
x_A+x_B+x_C =240,\\
60 \le x_A \le 140,\\
40 \le x_B \le 90,\\
20 \le x_C \le 40
\end{gather}
$$

を満たす配分の集合である．

### 三角形上でコアを理解する

3人の利得の合計が

$$
x_A+x_B+x_C=240
$$

に固定されているため，配分 $(x_A,x_B,x_C)$ は2次元の正三角形上の点として表せる．

- 頂点Aは $(240,0,0)$ を表し，Aが全額を受け取る
- 頂点Bは $(0,240,0)$ を表し，Bが全額を受け取る
- 頂点Cは $(0,0,240)$ を表し，Cが全額を受け取る
- 三角形の内部では，各頂点に近いほど対応するプレイヤーの利得が大きい
- ある頂点の反対側の辺と平行な直線上では，そのプレイヤーの利得が等しい

### 支配によるコアの説明

```{tip} 定義（提携による支配）
ある提携 $S$ について

$$
\sum_{i\in S}x_i<v(S)
$$

なる利得ベクトル $x$ について，次を満たす利得ベクトル $y$ が存在する．

$$
\sum_{i\in S}y_i\le v(S), \quad y_i>x_i \ \forall i\in S.
$$

このとき，$y$ は提携 $S$ を通じて $x$ を <span style="color:red">支配する</span> という．
```

コアはどの提携を通じても他の利得ベクトルに支配されない配分の集合と考えることもできる．

## コアが空になる場合

コアはつねに存在するとは限らない．
3人多数決ゲームを用いて，コアが空になる場合を確認する．

```{note} 演習4：3人多数決ゲームのコア
プレイヤー集合を

$$
N=\{1,2,3\}
$$

とする．2人以上が協力すれば意思決定でき，価値1を獲得できるとする．特性関数は，

$$
v(\{1\})=v(\{2\})=v(\{3\})=0,
$$

$$
v(\{1,2\})=v(\{1,3\})=v(\{2,3\})=1,
$$

$$
v(\{1,2,3\})=1
$$

である．コアに含まれる配分 $x=(x_1,x_2,x_3)$ が存在すると仮定し，次の問いに答えよ．

1. パレート最適性から，$x_1,x_2,x_3$ が満たす等式を書け．
2. 提携 $\{1,2\}$，$\{1,3\}$，$\{2,3\}$ の提携合理性を，それぞれ不等式で表せ．
3. 問2の3つの不等式の左辺と右辺を，それぞれすべて加えよ．
4. 問1の等式を問3の結果へ代入し，どのような矛盾が生じるか示せ．
5. このゲームのコアを答え，どの配分も安定しない理由を説明せよ．
```

```{dropdown} <span style="color:red">演習4 解答例</span>
**1．パレート最適性**

全員提携が生み出す価値は1なので，

$$
x_1+x_2+x_3=1
$$

である．

**2．2人提携の提携合理性**

各2人提携は価値1を獲得できるため，

$$
x_1+x_2\ge 1,
$$

$$
x_1+x_3\ge 1,
$$

$$
x_2+x_3\ge 1
$$

でなければならない．

**3．3つの不等式の和**

左辺では各プレイヤーの利得が2回ずつ現れるので，

$$
2x_1+2x_2+2x_3\ge 3
$$

すなわち，

$$
2(x_1+x_2+x_3)\ge 3
$$

となる．

**4．矛盾**

問1の $x_1+x_2+x_3=1$ を代入すると，

$$
2\ge 3
$$

となる．これは成立しないため，コアに含まれる配分が存在するという仮定と矛盾する．

**5．コア**

したがって，コアは

$$
\mathcal{C}(v)=\varnothing
$$

である．どの配分を選んでも，少なくとも1つの2人提携が，現在の配分より自分たちだけで価値1を分けたほうがよいと考える．
そのため，どの配分も提携による離脱に対して安定ではない．
```


```{warning} 課題1：3人対称ゲームのコア
プレイヤー集合を $N=\{1,2,3\}$ とし，特性関数を

$$
v(\{1\})=v(\{2\})=v(\{3\})=0,
$$

$$
v(\{1,2\})=v(\{1,3\})=v(\{2,3\})=a,
$$

$$
v(N)=1
$$

とする．ただし，$0<a<1$ とする．

1. コアに含まれる配分が満たす条件を書け．
2. 3つの2人提携の条件を加えて，コアが存在するためには $a\le 2/3$ が必要であることを示せ．
3. $a\le 2/3$ のとき，均等配分 $(1/3,1/3,1/3)$ がコアに含まれることを示せ．
4. コアが存在するための必要十分条件を答えよ．
```

```{dropdown} <span style="color:red">課題1 解答例</span>
**1．コアの条件**

コアに含まれる配分 $x=(x_1,x_2,x_3)$ は，

$$
x_1+x_2+x_3=1
$$

および

$$
x_1+x_2\ge a,\qquad
x_1+x_3\ge a,\qquad
x_2+x_3\ge a,\qquad
x_1,x_2,x_3\ge 0
$$

を満たす．

**2．必要条件**

3つの2人提携の条件を加えると，

$$
2(x_1+x_2+x_3)\ge 3a
$$

である．$x_1+x_2+x_3=1$ より，

$$
2\ge 3a
$$

でなければならない．したがって，

$$
a\le\frac{2}{3}
$$

はコアが存在するための必要条件である．

**3．均等配分**

均等配分では，任意の2人提携が受け取る利得の合計は

$$
\frac13+\frac13=\frac23
$$

である．$a\le 2/3$ ならば，

$$
\frac23\ge a
$$

なので，すべての2人提携について提携合理性を満たす．個人合理性とパレート最適性も満たすため，均等配分はコアに含まれる．

**4．必要十分条件**

$a\le 2/3$ は必要であり，この条件のもとでは実際に均等配分がコアに含まれる．したがって，コアが存在するための必要十分条件は

$$
a\le\frac23
$$

である．
```

## シャープレイ値

コアは安定な配分の集合を示すが，1つの配分に決まるとは限らず，空集合になることもある．
これに対して，シャープレイ値は各プレイヤーの貢献を平均することによって，つねに1つの配分を定める．

### 限界貢献度

プレイヤーが1人ずつ順番に提携へ参加すると考える．
ある提携 $S$ に最後にプレイヤー $i$ が参加したとき，その参加によって提携の価値がどれだけ増えたかを測る．

```{tip} 定義（限界貢献度）
プレイヤー $i\in S$ が提携 $S$ に参加することによる価値の増加

$$
v(S)-v(S\setminus\{i\})
$$

を，プレイヤー $i$ の提携 $S$ に対する <span style="color:red">限界貢献度</span> という．
```

### 具体例：ベンチャー企業の限界貢献度

A，B，Cがこの順で参加する場合を考える．

- Aの限界貢献度：$v(\{A\})-v(\varnothing)=60$
- Bの限界貢献度：$v(\{A,B\})-v(\{A\})=200-60=140$
- Cの限界貢献度：$v(N)-v(\{A,B\})=240-200=40$

参加順序は全部で $3!=6$ 通りある．各順序における限界貢献度は次のとおりである．

| 参加順序 | A | B | C |
|---|---:|---:|---:|
| A→B→C | 60 | 140 | 40 |
| A→C→B | 60 | 90 | 90 |
| B→A→C | 160 | 40 | 40 |
| B→C→A | 140 | 40 | 60 |
| C→A→B | 130 | 90 | 20 |
| C→B→A | 140 | 80 | 20 |
| **平均** | **115** | **80** | **45** |

各行の限界貢献度の合計は240であり，全員提携の価値に等しい．

```{tip} 定義（シャープレイ値）
すべての参加順序が同じ確率で起こると考えたときの，各プレイヤーの限界貢献度の平均を <span style="color:red">シャープレイ値</span> という．

プレイヤー $i$ のシャープレイ値を $\phi_i(v)$ とすると，

$$
\phi_i(v)
=\sum_{\substack{S\subseteq N\\ i\in S}}
\frac{(|S|-1)!(n-|S|)!}{n!}
\left[v(S)-v(S\setminus\{i\})\right]
$$

である．
```

ベンチャー企業の起業ゲームのシャープレイ値は，

$$
\phi(v)=(115,80,45)
$$

である．

### シャープレイ値の性質

シャープレイ値は，次の4つの性質によって特徴づけられる．

1. **パレート最適性**：シャープレイ値の合計は全員提携の価値に等しい

   $$
   \sum_{i\in N}\phi_i(v)=v(N)
   $$

2. **ナルプレイヤー**：どの提携に参加しても価値を増やさないプレイヤーのシャープレイ値は0である
3. **対称性**：すべての提携に対する限界貢献度が等しい2人のシャープレイ値は等しい
4. **加法性**：2つのゲームを足したゲームのシャープレイ値は，各ゲームのシャープレイ値の和に等しい

```{tip} 定義（ナルプレイヤー）
すべての $S\subseteq N\setminus\{i\}$ について

$$
v(S\cup\{i\})=v(S)
$$

を満たすプレイヤー $i$ を <span style="color:red">ナルプレイヤー</span> という．
```

### コアとシャープレイ値の違い

ベンチャー企業の起業ゲームのコアでは，Cの利得は

$$
20\le x_C\le 40
$$

でなければならない．しかし，シャープレイ値では

$$
\phi_C(v)=45
$$

である．したがって，このゲームのシャープレイ値 $(115,80,45)$ はコアに含まれない．

実際，AとBの利得の合計は

$$
115+80=195<200=v(\{A,B\})
$$

であるため，提携 $\{A,B\}$ はシャープレイ値による配分から離脱する誘因をもつ．

- **コア**は，どの提携も離脱したくならない安定性を重視する
- **シャープレイ値**は，参加順序に関する平均的な貢献を重視する

このため，両者は一致することもあるが，一般には異なる．

```{note} 演習5：多数決ゲームのシャープレイ値
3人多数決ゲームを考える．1人提携の価値は0，2人以上の提携の価値は1である．

1. 各プレイヤーが正の限界貢献度をもつのは，参加順序の何番目に参加するときか．
2. 6通りの参加順序を用いて，各プレイヤーのシャープレイ値を求めよ．
3. シャープレイ値はコアに含まれるか説明せよ．
```

```{dropdown} <span style="color:red">演習5 解答例</span>
**1．正の限界貢献度をもつ場合**

最初の1人だけでは価値は0であり，2人目が参加したときに提携の価値が0から1へ増える．3人目が参加しても価値は増えない．
したがって，参加順序の2番目になったプレイヤーだけが限界貢献度1をもつ．

**2．シャープレイ値**

各プレイヤーは，6通りの参加順序のうち2通りで2番目になる．したがって，各プレイヤーのシャープレイ値は

$$
\frac{2}{6}\cdot 1=\frac13
$$

である．よって，

$$
\phi(v)=\left(\frac13,\frac13,\frac13\right)
$$

となる．

**3．コアとの比較**

多数決ゲームのコアは空集合である．したがって，シャープレイ値はコアに含まれない．
シャープレイ値はコアが空であっても定義できる．
```

```{warning} 課題2：コアとシャープレイ値
プレイヤー集合を $N=\{1,2,3\}$ とし，特性関数を

$$
v(\{1\})=v(\{2\})=v(\{3\})=0,
$$

$$
v(\{1,2\})=6,\qquad
v(\{2,3\})=4,\qquad
v(\{1,3\})=3,
$$

$$
v(N)=10
$$

とする．

1. コアに含まれる配分が満たす等式と不等式を書け．
2. 全員提携の条件を用いて，コアを各プレイヤーの利得の範囲で表せ．
3. 各プレイヤーのシャープレイ値を求めよ．
4. シャープレイ値がコアに含まれるか確認せよ．
```

```{dropdown} <span style="color:red">課題2 解答例</span>
**1．コアの条件**

コアの配分 $x=(x_1,x_2,x_3)$ は，

$$
x_1+x_2+x_3=10
$$

および

$$
x_1+x_2\ge 6,\qquad
x_2+x_3\ge 4,\qquad
x_1+x_3\ge 3,
$$

$$
x_1,x_2,x_3\ge 0
$$

を満たす．

**2．コアの範囲**

全員提携の条件を使うと，

$$
x_1+x_2\ge 6\quad\Longleftrightarrow\quad x_3\le 4,
$$

$$
x_2+x_3\ge 4\quad\Longleftrightarrow\quad x_1\le 6,
$$

$$
x_1+x_3\ge 3\quad\Longleftrightarrow\quad x_2\le 7
$$

である．したがって，コアは

$$
x_1+x_2+x_3=10,\qquad
0\le x_1\le 6,\quad
0\le x_2\le 7,\quad
0\le x_3\le 4
$$

を満たす配分の集合である．

**3．シャープレイ値**

プレイヤー1について，

$$
\phi_1
=\frac16\{6+3\}+\frac13(10-4)
=\frac{9}{6}+2
=\frac72
$$

である．同様に，

$$
\phi_2
=\frac16\{6+4\}+\frac13(10-3)
=4,
$$

$$
\phi_3
=\frac16\{3+4\}+\frac13(10-6)
=\frac52
$$

である．したがって，

$$
\phi(v)=\left(\frac72,4,\frac52\right)
$$

となる．

**4．コアとの比較**

$$
\frac72+4+\frac52=10
$$

であり，

$$
\frac72+4=\frac{15}{2}\ge 6,\qquad
4+\frac52=\frac{13}{2}\ge 4,\qquad
\frac72+\frac52=6\ge 3
$$

を満たす．個人合理性も満たすため，このゲームのシャープレイ値はコアに含まれる．
```

## コアと戦略的交渉

協力ゲーム理論では，コアやシャープレイ値がどのような性質をもつかを公理的に考えた．
一方，誰が提案し，誰が受諾するかという交渉過程を展開型ゲームとして記述する方法を**戦略的アプローチ**という．

### ランダムな提案者ルール

3人対称ゲームを考える．1人提携の価値は0，2人提携の価値は $a$，全員提携の価値は1である．

各交渉ラウンドでは，次の手順で交渉する．

1. 3人から等確率で提案者を1人選ぶ
2. 提案者は，形成する提携とその利得配分を提案する
3. 提携に含まれる他のプレイヤーが順番に受諾または拒否する
4. 全員が受諾すれば提携が形成され，誰かが拒否すれば次のラウンドへ進む

将来利得に対する割引因子を $\delta$ とする．
定常的な部分ゲーム完全均衡におけるプレイヤー $i$ の期待利得を $w_i$ とする．

ゲームが対称であり，提案者が等確率で選ばれるなら，

$$
w_1=w_2=w_3=\frac13
$$

となる．

提案者が全員提携をつくる場合，2人の応答者にそれぞれ継続価値 $\delta/3$ を与えればよい．
したがって，提案者の利得は

$$
1-\frac{2\delta}{3}
$$

である．一方，2人提携をつくる場合，1人の応答者へ $\delta/3$ を与えるので，提案者の利得は

$$
a-\frac{\delta}{3}
$$

である．

したがって，全員提携が提案される条件は

$$
1-\frac{2\delta}{3}
\ge
a-\frac{\delta}{3}
$$

すなわち，

$$
a\le 1-\frac{\delta}{3}
$$

である．

プレイヤーが将来を十分に重視し，$\delta\to 1$ とすると，この条件は

$$
a\le\frac23
$$

に近づく．これは，課題1で求めた3人対称ゲームのコアが存在する条件と一致する．

この例では，コアが示す提携の安定性が，具体的な交渉ゲームの均衡としても解釈できる．

```{note} 演習6：全員提携が形成される条件
3人対称ゲームのランダムな提案者ルールを考え，割引因子を $\delta=0.9$ とする．

1. 全員提携を提案したときの提案者の利得を求めよ．
2. 2人提携の価値が $a=0.68$ のとき，2人提携を提案した場合の提案者の利得を求めよ．
3. $a=0.68$ のとき，提案者は全員提携と2人提携のどちらを選ぶか．
4. $a=0.75$ の場合はどちらを選ぶか．
```

```{dropdown} <span style="color:red">演習6 解答例</span>
**1．全員提携**

$$
1-\frac{2\delta}{3}
=1-\frac{2\cdot 0.9}{3}
=0.4
$$

である．

**2．2人提携**

$$
a-\frac{\delta}{3}
=0.68-\frac{0.9}{3}
=0.38
$$

である．

**3．$a=0.68$ の場合**

$$
0.4>0.38
$$

なので，提案者は全員提携を選ぶ．

**4．$a=0.75$ の場合**

2人提携を提案した場合の利得は

$$
0.75-\frac{0.9}{3}=0.45
$$

である．

$$
0.45>0.4
$$

なので，提案者は2人提携を選ぶ．
```

## 応用：市場ゲーム

協力ゲームは，市場における取引の分析にも利用できる．

```{tip} 定義（市場ゲーム）
市場参加者が形成できる提携と，各提携が取引によって獲得できる価値を特性関数で表したゲームを <span style="color:red">市場ゲーム</span> という．
```

### 具体例：1人の売り手と2人の買い手

学生Aは自転車を1台もっており，その評価額は5000円である．
学生BとCは自転車を買いたいと考えており，評価額はそれぞれ6000円と7000円である．

金額の単位を1000円とすると，市場ゲームの特性関数は，

$$
v(\{A\})=5,\qquad v(\{B\})=v(\{C\})=0,
$$

$$
v(\{A,B\})=6,\qquad
v(\{A,C\})=7,\qquad
v(\{B,C\})=0,
$$

$$
v(\{A,B,C\})=7
$$

である．

各プレイヤーが単独で得られる価値を差し引き，協力によって新たに生まれる余剰だけを表すと，

$$
\widehat v(S)=v(S)-\sum_{i\in S}v(\{i\})
$$

となる．

```{tip} 定義（ゼロ正規化）
各プレイヤーの1人提携の価値が0になるように特性関数を変換することを <span style="color:red">ゼロ正規化</span> という．
```

ゼロ正規化した市場ゲームは，

$$
\widehat v(\{A\})=\widehat v(\{B\})=\widehat v(\{C\})=0,
$$

$$
\widehat v(\{A,B\})=1,\qquad
\widehat v(\{A,C\})=2,\qquad
\widehat v(\{B,C\})=0,
$$

$$
\widehat v(N)=2
$$

である．

コアに含まれる余剰配分 $x=(x_A,x_B,x_C)$ は，

$$
x_A+x_B+x_C=2,
$$

$$
x_A+x_B\ge 1,\qquad
x_A+x_C\ge 2,\qquad
x_B+x_C\ge 0,
$$

$$
x_A,x_B,x_C\ge 0
$$

を満たす．

$x_A+x_C\ge 2$ と $x_A+x_B+x_C=2$ から $x_B=0$ である．
したがって，

$$
x_A+x_C=2,\qquad 1\le x_A\le 2,\qquad x_B=0
$$

となる．

Aの余剰 $x_A$ は，自転車に対する評価額5000円を超えて受け取る金額である．
したがって，取引価格 $p$ は

$$
p=5000+1000x_A
$$

と表され，コアが示す価格の範囲は

$$
6000\le p\le 7000
$$

円である．

これは，最も高い評価額をもつCが自転車を購入し，価格が2人の買い手の評価額6000円と7000円の間に決まることを示している．

---

## まとめ

- **特性関数形ゲーム**は，プレイヤー集合 $N$ と各提携の価値を表す特性関数 $v$ の組である．
- **配分**は，パレート最適性と個人合理性を満たす利得ベクトルである．
- **コア**は，すべての提携について提携合理性を満たし，どの提携も離脱したくならない配分の集合である．
- コアは複数の配分を含む場合も，空集合になる場合もある．
- **シャープレイ値**は，すべての参加順序における限界貢献度の平均であり，つねに1つの配分を定める．
- コアは提携の安定性，シャープレイ値は平均的な貢献を重視するため，一般には一致しない．
- 市場ゲームでは，コアによって競争的な取引価格の範囲を説明できる．

## 自主学習用の発展問題

```{note} 発展問題1：1人の経営者と2人の労働者
プレイヤー1を経営者，プレイヤー2と3を労働者とする．
経営者が労働者を1人だけ雇用すると企業の収益は80万円，2人とも雇用すると100万円である．
経営者だけ，労働者だけ，または労働者2人だけでは収益を得られない．

1. 特性関数をすべて書け．
2. コアを求めよ．
3. シャープレイ値を求めよ．
4. シャープレイ値がコアに含まれるか確認せよ．
```

```{dropdown} 発展問題1 解答例
**1．特性関数**

$$
v(\{1\})=v(\{2\})=v(\{3\})=0,
$$

$$
v(\{1,2\})=v(\{1,3\})=80,\qquad
v(\{2,3\})=0,
$$

$$
v(\{1,2,3\})=100
$$

である．

**2．コア**

コアの配分は，

$$
x_1+x_2+x_3=100,
$$

$$
x_1+x_2\ge 80,\qquad
x_1+x_3\ge 80,\qquad
x_2+x_3\ge 0,
$$

$$
x_1,x_2,x_3\ge 0
$$

を満たす．全員提携の条件から，

$$
x_3\le 20,\qquad x_2\le 20
$$

である．したがって，コアは

$$
x_1=100-x_2-x_3,\qquad
0\le x_2\le 20,\qquad
0\le x_3\le 20
$$

を満たす配分の集合である．このとき，自動的に $60\le x_1\le 100$ となる．

**3．シャープレイ値**

プレイヤー1について，

$$
\phi_1
=\frac16(80+80)+\frac13(100-0)
=60
$$

である．プレイヤー2について，

$$
\phi_2
=\frac16(80+0)+\frac13(100-80)
=20
$$

である．対称性から $\phi_3=20$ なので，

$$
\phi(v)=(60,20,20)
$$

となる．

**4．コアとの比較**

シャープレイ値は

$$
60+20=80,\qquad 60+20=80
$$

を満たし，個人合理性とパレート最適性も満たす．したがって，このゲームのシャープレイ値はコアに含まれる．
```

```{note} 発展問題2：右手と左手の手袋
プレイヤー1は右手の手袋を1つ，プレイヤー2と3は左手の手袋を1つずつもっている．
右手と左手の手袋が1組できる提携の価値を1，それ以外の提携の価値を0とする．

1. 特性関数をすべて書け．
2. コアを求めよ．
3. シャープレイ値を求めよ．
4. コアとシャープレイ値の違いを説明せよ．
```

```{dropdown} 発展問題2 解答例
**1．特性関数**

$$
v(\{1\})=v(\{2\})=v(\{3\})=0,
$$

$$
v(\{1,2\})=v(\{1,3\})=1,\qquad
v(\{2,3\})=0,
$$

$$
v(\{1,2,3\})=1
$$

である．

**2．コア**

コアの条件は，

$$
x_1+x_2+x_3=1,
$$

$$
x_1+x_2\ge 1,\qquad
x_1+x_3\ge 1,\qquad
x_2+x_3\ge 0,
$$

$$
x_1,x_2,x_3\ge 0
$$

である．最初の2つの提携合理性から，

$$
x_3\le 0,\qquad x_2\le 0
$$

である．非負条件より $x_2=x_3=0$ であり，$x_1=1$ となる．したがって，

$$
\mathcal{C}(v)=\{(1,0,0)\}
$$

である．

**3．シャープレイ値**

プレイヤー1は，6通りの参加順序のうち4通りで価値を0から1へ増やす．
プレイヤー2と3は，それぞれ1通りで価値を0から1へ増やす．したがって，

$$
\phi(v)=\left(\frac23,\frac16,\frac16\right)
$$

である．

**4．コアとの比較**

シャープレイ値では，

$$
\phi_1+\phi_2=\frac56<1=v(\{1,2\})
$$

である．したがって，提携 $\{1,2\}$ が離脱する誘因をもち，シャープレイ値はコアに含まれない．

コアは希少な右手の手袋をもつプレイヤー1へ価値をすべて配分する．一方，シャープレイ値は参加順序によって左手の手袋も価値を生む場合があることを反映する．
```

```{note} 発展問題3：市場ゲームと取引価格
売り手Aが自転車を1台もち，その評価額は4000円である．
買い手BとCの評価額は，それぞれ6000円と9000円である．金額の単位を1000円とする．

1. ゼロ正規化した特性関数を求めよ．
2. コアを求め，コアが示す取引価格の範囲を答えよ．
3. シャープレイ値を求めよ．
4. シャープレイ値がコアに含まれるか確認せよ．
```

```{dropdown} 発展問題3 解答例
**1．ゼロ正規化した特性関数**

協力によって生まれる余剰を考えると，

$$
\widehat v(\{A\})=\widehat v(\{B\})=\widehat v(\{C\})=0,
$$

$$
\widehat v(\{A,B\})=6-4=2,\qquad
\widehat v(\{A,C\})=9-4=5,\qquad
\widehat v(\{B,C\})=0,
$$

$$
\widehat v(N)=9-4=5
$$

である．

**2．コアと取引価格**

コアの条件は，

$$
x_A+x_B+x_C=5,
$$

$$
x_A+x_B\ge 2,\qquad
x_A+x_C\ge 5,\qquad
x_B+x_C\ge 0,
$$

$$
x_A,x_B,x_C\ge 0
$$

である．$x_A+x_C\ge 5$ と全員提携の条件から $x_B=0$ である．したがって，

$$
x_A+x_C=5,\qquad 2\le x_A\le 5,\qquad x_B=0
$$

となる．取引価格は

$$
p=4000+1000x_A
$$

なので，

$$
6000\le p\le 9000
$$

円である．

**3．シャープレイ値**

$$
\phi_A
=\frac16(2+5)+\frac13(5-0)
=\frac{17}{6},
$$

$$
\phi_B
=\frac16(2+0)+\frac13(5-5)
=\frac13,
$$

$$
\phi_C
=\frac16(5+0)+\frac13(5-2)
=\frac{11}{6}
$$

である．したがって，

$$
\phi(v)=\left(\frac{17}{6},\frac13,\frac{11}{6}\right)
$$

となる．

**4．コアとの比較**

コアでは $x_B=0$ でなければならないが，シャープレイ値では $\phi_B=1/3>0$ である．また，

$$
\phi_A+\phi_C
=\frac{28}{6}
=\frac{14}{3}
<5
=\widehat v(\{A,C\})
$$

なので，提携 $\{A,C\}$ が離脱する誘因をもつ．したがって，シャープレイ値はコアに含まれない．
```

## Pythonコード

ベンチャー企業の起業ゲームについて，すべての参加順序からシャープレイ値を計算し，指定した利得ベクトルがコアに含まれるか判定する．

````{dropdown} シャープレイ値とコアを計算するプログラム
```python
from itertools import combinations, permutations

players = ("A", "B", "C")

v = {
    frozenset(): 0,
    frozenset({"A"}): 60,
    frozenset({"B"}): 40,
    frozenset({"C"}): 20,
    frozenset({"A", "B"}): 200,
    frozenset({"A", "C"}): 150,
    frozenset({"B", "C"}): 100,
    frozenset({"A", "B", "C"}): 240,
}


def shapley_value(players, characteristic_function):
    """すべての参加順序における限界貢献度の平均を求める．"""
    total = {player: 0.0 for player in players}
    orders = list(permutations(players))

    for order in orders:
        coalition = frozenset()

        for player in order:
            new_coalition = coalition | {player}
            marginal_contribution = (
                characteristic_function[new_coalition]
                - characteristic_function[coalition]
            )
            total[player] += marginal_contribution
            coalition = new_coalition

    return {
        player: total[player] / len(orders)
        for player in players
    }


def proper_coalitions(players):
    """空提携と全員提携を除く提携を列挙する．"""
    for size in range(1, len(players)):
        for members in combinations(players, size):
            yield frozenset(members)


def check_core(allocation, players, characteristic_function, tolerance=1e-9):
    """配分がコアに含まれるかを判定する．"""
    grand_coalition = frozenset(players)
    allocated_total = sum(allocation[player] for player in players)

    if abs(allocated_total - characteristic_function[grand_coalition]) > tolerance:
        return False, "全員提携の価値を過不足なく配分していない"

    for coalition in proper_coalitions(players):
        coalition_payoff = sum(allocation[player] for player in coalition)

        if coalition_payoff + tolerance < characteristic_function[coalition]:
            return False, coalition

    return True, None


phi = shapley_value(players, v)
print("シャープレイ値:", phi)

core_candidate = {"A": 110, "B": 90, "C": 40}
print("配分 (110, 90, 40):", check_core(core_candidate, players, v))

shapley_allocation = phi
print("シャープレイ値:", check_core(shapley_allocation, players, v))
```
````

このプログラムでは，`permutations` を用いてすべての参加順序を列挙している．
また，`check_core` は，全員提携の価値が過不足なく配分されているかを確認した後，すべての1人提携と2人提携について提携合理性を調べている．

実行すると，シャープレイ値は

```text
{'A': 115.0, 'B': 80.0, 'C': 45.0}
```

となる．配分 $(110,90,40)$ はコアに含まれるが，シャープレイ値は提携 $\{A,B\}$ の条件を満たさないことも確認できる．
