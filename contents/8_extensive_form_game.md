## 展開型ゲーム

- 戦略型ゲーム：誰が何を選べるかと，その結果としてどの利得が得られるかを利得表で表す．
- **展開型ゲーム**：プレイヤーがどの順序で意思決定するかを明示したゲーム．
意思決定の順序を**ゲーム木**として表す．

```{tip} 定義（展開型ゲーム）
<span style="color:red">展開型ゲーム</span>とは，プレイヤーの意思決定の順序，その時点で選べる行動，最終的な利得を木構造で表したゲームである．
```

例）
- 企業が市場に参入するかどうかを先に決める
- 既存企業が，参入を見てから対抗するか受け入れるかを決める
- 交渉で，一方が提案し，他方が受け入れるか拒否するかを決める
- 政策当局が規制するかどうかを決め，企業がその後の行動を選ぶ

展開型ゲームでは**いつ選ぶか**の要素が関わってくる．

---

## ゲーム木

ゲーム木は次の要素から構成される．
- **意思決定点**：プレイヤーが行動を選ぶ点．
各意思決定点ではどのプレイヤーが行動するかが決まっている．
- **枝**：その意思決定点で選べる行動．
意思決定点から伸びる線で表す．
- **終点**：それ以上行動が選ばれない点．
終点には各プレイヤーの利得が割り当てられる．
- **履歴**：初期点からある点までに選ばれた行動の列．
展開型ゲームでは，ある履歴の後にどの行動を選ぶかが重要になる．

```{tip} 定義：有限展開型ゲーム
構成するゲーム木が有限グラフ（頂点と辺の数が有限）である展開型ゲームを**有限展開型ゲーム**とよぶ．
```

### 例：参入ゲーム

次の参入ゲームを考える．

- プレイヤー1：新規企業，プレイヤー2：既存企業
- 新規企業は，最初に「参入する」か「参入しない」かを選ぶ
- 新規企業が参入した場合，既存企業は「受け入れる」か「対抗する」かを選ぶ
- 新規企業が参入しない場合，そこでゲームは終わる

プレイヤー1・2の利得を次のようにする．

- 参入しない：$(1,5)$
- 参入する，受け入れる：$(3,3)$
- 参入する，対抗する：$(0,1)$

このゲームは次のようなゲーム木で表せる．

```text
新規企業
├─ 参入しない -> (1, 5)
└─ 参入する
   └─ 既存企業
      ├─ 受け入れる -> (3, 3)
      └─ 対抗する -> (0, 1)
```

このゲームでは新規企業が先に行動し，参入した場合だけ既存企業が行動する．

各構成要素は次のようになる．
- **意思決定点**：新規企業が意思決定する時点，既存企業が意思決定する時点
- **枝**：新規企業が意思決定する時点では「参入する」「参入しない」，既存企業が意思決定する時点では「受け入れる」「対抗する」
- **終点**：参入しない，参入する後に受け入れる，参入する後に対抗する

````{note} 演習1：ゲーム木の読み取り
   └─ プレイヤー2

2. プレイヤー2の行動は何か．
3. 終点はいくつあるか．
````

## 完全情報ゲームと情報集合

展開型ゲームでは，後から行動するプレイヤーが，それ以前にどの行動が選ばれたかを知っているかどうかによって分類される．

```{tip} 定義（完全情報ゲーム）
ある展開型ゲームが<span style="color:red">完全情報ゲーム</span>であるとは，各プレイヤーが自分の意思決定点に到達したとき，それ以前に選ばれた行動をすべて観察できることをいう．
```

プレイヤーが自分のいる意思決定点を区別できない場合には**情報集合**を用いる．

```{tip} 定義（情報集合）
プレイヤーが区別できない意思決定点の集合を<span style="color:red">情報集合</span>という．
同じ情報集合に属する意思決定点では，そのプレイヤーは同じ行動集合を持ち，同じ行動を選ばなければならない．

※ 行動集合は次節で定義する．
```

完全情報ゲームでない，不完全情報を含むゲームは第10回の情報不完備ゲームで改めて扱う．

## 展開型ゲームにおける戦略

戦略型ゲームでは戦略はプレイヤーが選ぶ行動として理解できたが，展開型ゲームでは，**戦略**と**行動**を区別する必要がある．

展開型ゲームにおける戦略は，**自分が行動する可能性のあるすべての意思決定点で，どの行動を選ぶかを指定した計画**である．

```{tip} 定義（展開型ゲームにおける戦略）
展開型ゲームにおけるプレイヤーの戦略は，そのプレイヤーが行動する可能性のあるすべての意思決定点に対して，そこで選ぶ行動をあらかじめ指定した行動計画を指す．
```

参入ゲームでは，新規企業は最初の意思決定点で1回だけ行動する．
したがって，新規企業の戦略集合は

$$
S_1=\{\text{参入する},\text{参入しない}\}
$$

である．

既存企業は，新規企業が参入した場合に1回だけ行動する．
したがって，既存企業の戦略集合は

$$
S_2=\{\text{受け入れる},\text{対抗する}\}
$$

である．

一般に1人のプレイヤーが複数の意思決定点を持つ場合には，戦略はそれぞれの場面で何をするかをすべて指定する．

例えば次の展開型ゲームを考える．
プレイヤー$i$の戦略集合を$S_i$とおく．


```text
プレイヤー1
├─ A
│  └─ プレイヤー2
│     ├─ C -> (3, 1)
│     └─ D -> (0, 2)
└─ B
   └─ プレイヤー2
      ├─ E -> (1, 3)
      └─ F -> (2, 0)
```

プレイヤー1は最初に $A$ または $B$ を選ぶため

$$
S_1=\{A,B\}
$$

であり，プレイヤー2はプレイヤー1が$A$，$B$のどちらの行動を選択するかによって取る行動が変わるため

$$
(A\text{の後の行動},B\text{の後の行動})
$$

として表される．
したがって

$$
S_2=\{(C,E),(C,F),(D,E),(D,F)\}
$$

となる．

```{tip} 注意
プレイヤー2の戦略は実際に到達した意思決定点で何をするかではなく，**到達しうる全ての意思決定点で何をする予定か**を網羅的に指定する．
```

## 後ろ向き帰納法

**後ろ向き帰納法**：完全情報の有限展開型ゲームにおいて，ゲームの最後の意思決定点から順に最適行動を求める方法

```{tip} 定義（後ろ向き帰納法）
ゲーム木の終点に近い意思決定点から順に，各プレイヤーにとって最適な行動を選び，その結果を前の意思決定点に戻していく解法を<span style="color:red">後ろ向き帰納法</span>という．
```

参入ゲームで後ろ向き帰納法を用いる．

```text
新規企業
├─ 参入しない -> (1, 5)
└─ 参入する
   └─ 既存企業
      ├─ 受け入れる -> (3, 3)
      └─ 対抗する -> (0, 1)
```

1. 新規企業が参入した後の既存企業の意思決定を考える．

    - 受け入れる：既存企業の利得は $3$
    - 対抗する：既存企業の利得は $1$

    → 既存企業は受け入れる方がよい．

2. 新規企業の意思決定を考える．
1より，新規企業は参入すれば既存企業が受け入れると予想する．

    - 参入しない：新規企業の利得は $1$
    - 参入する：新規企業の利得は $3$

    → 新規企業は参入する方がよい．

以上より，後ろ向き帰納法による結果は

$$
(\text{参入する},\text{受け入れる})
$$

であり，実現する利得は

$$
(3,3)
$$

である．


## 戦略型ゲームへの変換

展開型ゲームは各プレイヤーの戦略をすべて列挙すれば**戦略型ゲーム**として表せる．
参入ゲームを戦略型ゲームとして表すと，次の利得表になる．

| 新規企業＼既存企業 | 受け入れる | 対抗する |
|---|:---:|:---:|
| 参入する | $(3,3)$ | $(0,1)$ |
| 参入しない | $(1,5)$ | $(1,5)$ |

新規企業が参入しない場合には，既存企業の行動は実際には観察されないが，戦略型ゲームに変換するにあたっては既存企業がもし参入されたらどうするかという行動計画を持っているものとして表す．

この戦略型ゲームのナッシュ均衡は

$$
(\text{参入する},\text{受け入れる}), (\text{参入しない},\text{対抗する})
$$

である．

## 信頼できない脅し

戦略型表現でナッシュ均衡であった戦略プロフィール $(\text{参入しない},\text{対抗する})$ は，既存企業が「参入してきたら対抗する」と脅すことで，新規企業が参入しない結果によって実現される．

しかし，実際に新規企業が参入した後を考えると，既存企業にとっては対抗するより受け入れる方が高い利得を与える．

$$
3>1
$$

したがって，既存企業の「対抗する」という脅しは，実際にその場面になれば実行されない．
このような脅しを**信頼できない脅し**という．

```{tip} 定義（信頼できない脅し）
ゲームのある時点では実行することが最適でないにもかかわらず，相手の行動を変えさせるために事前に使われる脅しを<span style="color:red">信頼できない脅し</span>という．
```

## 部分ゲームと部分ゲーム完全均衡

**部分ゲーム**：展開型ゲームにおいてゲームの途中から始まる小さなゲーム

```{tip} 定義（部分ゲーム）
展開型ゲームにおいて，ある意思決定点から始まり，それ以降のすべての枝と終点を含むゲームを部分ゲームという．
```

参入ゲームでは新規企業が参入した後に既存企業が行動する部分が部分ゲームと考えられる．

```text
既存企業
├─ 受け入れる -> (3, 3)
└─ 対抗する -> (0, 1)
```

**部分ゲーム完全均衡**：ゲーム全体を含むすべての部分ゲームでナッシュ均衡になっている戦略プロフィール

```{tip} 定義（部分ゲーム完全均衡）
展開型ゲームにおいて，すべての部分ゲームでナッシュ均衡になっている戦略プロフィールを<span style="color:red">部分ゲーム完全均衡</span>という．
```

完全情報の有限展開型ゲームでは，後ろ向き帰納法で得られる解が部分ゲーム完全均衡である．

参入ゲームでは，

$$
(\text{参入する},\text{受け入れる})
$$

が部分ゲーム完全均衡である．  
一方，参入後の部分ゲームでは既存企業にとって対抗することは最適ではないため，

$$
(\text{参入しない},\text{対抗する})
$$

はナッシュ均衡ではあるが，部分ゲーム完全均衡ではない．

```{tip} 定理：ナッシュ均衡と部分ゲーム完全均衡の包含関係
$$
\text{部分ゲーム完全均衡}
\Rightarrow
\text{ナッシュ均衡}
$$

であり，一般に逆は成り立たない．
```

```{dropdown} <span style="color:red">課題1 解答例</span>
プレイヤー2が行動する場面では，$C$ を選ぶと利得は $1$，$D$ を選ぶと利得は $2$ である．
したがって，プレイヤー2は $D$ を選ぶ．
これを予想すると，プレイヤー1は $A$ の利得 $1$ と $B$ の後に $D$ が選ばれる場合の利得 $2$ を比較し，$B$ を選ぶ．

| 1＼2 | $C$ | $D$ |
|---|:---:|:---:|

BR_2(A)=\{C,D\},\qquad BR_2(B)=\{D\}

である．

後ろ向き帰納法で得られる $(B,D)$ が部分ゲーム完全均衡である．

したがって，信頼できない脅しを含む別のナッシュ均衡は存在しない．
```

## まとめ

- 展開型ゲーム：プレイヤーの意思決定の順序をゲーム木で表す
- ゲーム木：意思決定点・枝・終点・利得から構成される
- 展開型ゲームにおける戦略：行動する可能性のあるすべての意思決定点での行動計画
- 完全情報の有限展開型ゲームでは後ろ向き帰納法によって解を求められる
- 展開型ゲームは各プレイヤーの戦略を列挙すれば戦略型ゲームに変換できる
- ナッシュ均衡には信頼できない脅しを含むものがある．
- 部分ゲーム完全均衡はすべての部分ゲームで合理的な行動を要求する均衡概念である．

第9回は繰り返しゲームを扱う．  
同じゲームを何度も行う場合，現在の行動が将来の利得に影響するため，1回限りのゲームとは異なる行動が生じることがある．

## おまけ：Pythonで後ろ向き帰納法を確認する

参入ゲームを Python で確認する．
最後の意思決定点から順に最適行動を選ぶことで，後ろ向き帰納法を実装できる．

```python
payoffs = {
    "not_enter": (1, 5),
    "enter_accept": (3, 3),
    "enter_fight": (0, 1),
}

incumbent_actions = {
    "accept": payoffs["enter_accept"],
    "fight": payoffs["enter_fight"],
}

incumbent_choice = max(
    incumbent_actions,
    key=lambda action: incumbent_actions[action][1]
)

payoff_after_entry = incumbent_actions[incumbent_choice]

entrant_actions = {
    "not_enter": payoffs["not_enter"],
    "enter": payoff_after_entry,
}

entrant_choice = max(
    entrant_actions,
    key=lambda action: entrant_actions[action][0]
)

print("既存企業の選択:", incumbent_choice)
print("新規企業の選択:", entrant_choice)
print("結果の利得:", entrant_actions[entrant_choice])
```

このコードでは，まず既存企業の最適行動を求め，その結果を新規企業の意思決定に戻している．
