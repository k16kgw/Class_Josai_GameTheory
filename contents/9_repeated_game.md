### 前回の復習

- 展開型ゲームでは，プレイヤーの意思決定の順序をゲーム木で表した．

- 完全情報の有限展開型ゲームでは，後ろ向き帰納法によって解を求められる．
- ナッシュ均衡の中には，信頼できない脅しを含むものがある．
- 部分ゲーム完全均衡は，全ての部分ゲームで合理的な行動を要求する均衡概念である．

## 繰り返しゲームとは

第6回で，囚人のジレンマでは各プレイヤーが裏切りを選ぶことがナッシュ均衡になることを学んだ．
しかし，現実には，同じ相手と何度も関わる状況では協力が続くこともある．

例えば，次のような状況を考える．

- 企業同士が何度も価格競争を行う
- 友人同士が何度も助け合う
- ネットワーク参加者が繰り返し通信資源を利用する
- 地域住民が継続的に公共財の維持に関わる

このように，同じ構造のゲームが何度も繰り返される状況を**繰り返しゲーム**という．

```{tip} 定義（繰り返しゲーム）
ある基礎となるゲームを複数回繰り返すゲームを，繰り返しゲームという．
各回で行われる基礎となるゲームを，ステージゲームという．
```

繰り返しゲームでは，ある回の行動がその回の利得だけでなく，将来の相手の行動にも影響する．
そのため，1回限りのゲームとは異なる行動が合理的になることがある．

---

## ステージゲームと履歴

繰り返しゲームでは，1回ごとに行われるゲームを**ステージゲーム**という．
例えば，繰り返し囚人のジレンマでは，次の囚人のジレンマが毎期行われる．

| 1＼2 | 協力 $C$ | 裏切り $D$ |
|------|----------|------------|
| 協力 $C$ | (3, 3) | (0, 4) |
| 裏切り $D$ | (4, 0) | (1, 1) |

各期の行動の列を**履歴**という．
例えば，2期目の前に

$$
(C,C)
$$

が観察されていれば，「1期目に両者が協力した」という履歴を持つ．
3期目の前には，1期目と2期目の行動の組合せが履歴になる．

```{tip} 定義（履歴）
繰り返しゲームにおいて，ある時点までに各プレイヤーが選んだ行動の列を履歴という．
```

繰り返しゲームにおける戦略は，単に「協力する」「裏切る」ではない．
過去の履歴を見て，その後に何をするかを決める行動計画である．

```{tip} 定義（繰り返しゲームにおける戦略）
繰り返しゲームにおける戦略とは，各時点の履歴に対して，その時点で選ぶ行動を指定する計画である．
```

## 有限回繰り返しゲーム

まず，ゲームが有限回だけ繰り返される場合を考える．
ここでは，囚人のジレンマが2回繰り返される状況を考える．

1回限りの囚人のジレンマでは，裏切り $D$ が各プレイヤーの支配戦略である．
したがって，1回限りなら $(D,D)$ がナッシュ均衡になる．

2回繰り返しゲームでは，最後の2期目から考える．
2期目はゲームの最後なので，その後の将来は存在しない．
したがって，2期目では1回限りの囚人のジレンマと同じく，両者は裏切りを選ぶ．

次に1期目を考える．
1期目に協力しても，2期目にはいずれにせよ裏切りが選ばれる．
したがって，1期目の協力は将来の協力を引き出す効果を持たない．
そのため，1期目でも裏切りが選ばれる．

このように，有限回繰り返し囚人のジレンマでは，最終期から後ろ向き帰納法を用いると，全ての期で裏切りが選ばれる．

```{note}
有限回繰り返しゲームでも，ステージゲームに複数の均衡がある場合や，情報が不完全な場合には，より複雑な協力が生じることがある．
ここでは，基本的な囚人のジレンマを完全情報で有限回繰り返す場合を考えている．
```

## 無限回繰り返しゲーム

次に，ゲームが無限回続く場合，またはいつ終わるか分からない場合を考える．

この場合，現在の行動は将来の相手の行動に影響する．
もし自分が裏切れば，今期は高い利得を得られるかもしれない．
しかし，将来，相手も裏切るようになれば，長期的には損をする可能性がある．

したがって，プレイヤーが将来を十分に重視するなら，協力が合理的になることがある．

---

## 割引因子

将来の利得をどの程度重視するかを表すために，**割引因子**を用いる．

```{tip} 定義（割引因子）
$0<\delta<1$ を割引因子という．
今期の利得をそのまま評価し，1期後の利得を $\delta$ 倍，2期後の利得を $\delta^2$ 倍して評価する．
```

利得列

$$
u_1,u_2,u_3,\ldots
$$

が得られるとき，割引現在価値は

$$
u_1+\delta u_2+\delta^2 u_3+\cdots
$$

である．

例えば，毎期同じ利得 $x$ を得るなら，割引現在価値は

$$
x+\delta x+\delta^2 x+\cdots=\frac{x}{1-\delta}
$$

である．

$\delta$ が大きいほど，プレイヤーは将来を重視している．
$\delta$ が小さいほど，プレイヤーは現在の利得を重視している．

## トリガー戦略

無限回繰り返し囚人のジレンマで協力を支える代表的な戦略に，**トリガー戦略**がある．

```{tip} 定義（トリガー戦略）
最初は協力する．
相手が一度でも裏切ったら，それ以降ずっと裏切る．
このような戦略をトリガー戦略という．
```

トリガー戦略では，裏切りに対して将来の罰が与えられる．
そのため，現在だけを見ると裏切りが得でも，将来の損失を考えると協力を続ける方がよい場合がある．

---

## 協力が維持される条件

次の囚人のジレンマを考える．

| 1＼2 | 協力 $C$ | 裏切り $D$ |
|------|----------|------------|
| 協力 $C$ | (3, 3) | (0, 4) |
| 裏切り $D$ | (4, 0) | (1, 1) |

両者がトリガー戦略を用いているとする．
協力を続ける場合，各期に利得 3 を得る．
したがって，協力を続ける利得の割引現在価値は

$$
3+\delta 3+\delta^2 3+\cdots=\frac{3}{1-\delta}
$$

である．

一方，今期だけ裏切ると，今期の利得は 4 になる．
しかし，次期以降は相手もずっと裏切るため，各期の利得は 1 になる．
したがって，裏切った場合の割引現在価値は

$$
4+\delta 1+\delta^2 1+\cdots
=4+\frac{\delta}{1-\delta}
$$

である．

協力が維持されるためには，協力を続ける利得が，今期だけ裏切る利得以上であればよい．

$$
\frac{3}{1-\delta}
\ge
4+\frac{\delta}{1-\delta}
$$

この不等式を解くと，

$$
\delta\ge \frac{1}{3}
$$

である．

つまり，プレイヤーが将来を十分に重視し，$\delta$ が $1/3$ 以上であれば，トリガー戦略によって協力が維持されうる．

## しっぺ返し戦略

繰り返しゲームでよく用いられるもう1つの戦略に，**しっぺ返し戦略**がある．

```{tip} 定義（しっぺ返し戦略）
最初は協力する．
2期目以降は，前期に相手が選んだ行動をそのまま選ぶ．
このような戦略をしっぺ返し戦略という．
```

しっぺ返し戦略は，相手が協力すれば協力し，相手が裏切れば次の期に裏切る．
また，相手が再び協力すれば，こちらも協力に戻る．

トリガー戦略は一度の裏切りに対して永続的に罰する戦略である．
一方，しっぺ返し戦略は，一度の裏切りに対して一時的に罰する戦略である．

## Pythonで繰り返し囚人のジレンマを確認する

次のコードは，いくつかの単純な戦略を使って繰り返し囚人のジレンマを実行する例である．

```python
PAYOFFS = {
    ("C", "C"): (3, 3),
    ("C", "D"): (0, 4),
    ("D", "C"): (4, 0),
    ("D", "D"): (1, 1),
}

def always_cooperate(my_history, other_history):
    return "C"

def always_defect(my_history, other_history):
    return "D"

def tit_for_tat(my_history, other_history):
    if not other_history:
        return "C"
    return other_history[-1]

def play_repeated(strategy1, strategy2, rounds=10):
    history1 = []
    history2 = []
    total1 = 0
    total2 = 0

    for _ in range(rounds):
        a1 = strategy1(history1, history2)
        a2 = strategy2(history2, history1)
        u1, u2 = PAYOFFS[(a1, a2)]
        total1 += u1
        total2 += u2
        history1.append(a1)
        history2.append(a2)

    return history1, history2, total1, total2

print(play_repeated(always_cooperate, always_defect, rounds=5))
print(play_repeated(tit_for_tat, always_defect, rounds=5))
print(play_repeated(tit_for_tat, tit_for_tat, rounds=5))
```

このコードでは，戦略が履歴に依存していることを確認できる．
繰り返しゲームでは，過去の行動に応じて現在の行動を変える点が重要である．

## 本日のまとめ

- 繰り返しゲームでは，同じステージゲームが複数回繰り返される．
- 繰り返しゲームにおける戦略は，履歴に応じて行動を指定する計画である．
- 有限回繰り返し囚人のジレンマでは，後ろ向き帰納法により全ての期で裏切りが選ばれる．
- 無限回または終了時点が不確実な繰り返しゲームでは，将来の罰や報酬によって協力が維持されることがある．
- 割引因子が大きいほど，将来を重視するため，協力が成立しやすくなる．
- トリガー戦略やしっぺ返し戦略は，繰り返し関係において協力を支える代表的な戦略である．
